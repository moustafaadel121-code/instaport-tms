# InstaPort TMS — دليل التسليم والتشغيل

## أ) اللي بيتسلّم للشركة (محتويات حزمة التسليم)

| الملف | الوصف |
|---|---|
| `dist/index.html` | التطبيق (نسخة مشفرة ومرخصة بدومين الشركة) |
| `dist/logo.png`, `dist/manifest.json`, `dist/sw.js` | ملفات الواجهة |
| `server.js` | السيرفر (Node.js) |
| `package.json` | التبعيات |
| `.env.example` | نموذج الإعدادات — الشركة تملأه بقيمها |
| `supabase-schema.sql` | سكيما قاعدة البيانات — تتنفذ مرة واحدة |
| `DELIVERY_GUIDE.md` | هذا الملف |

> ⚠️ **لا يتسلّم أبدًا**: `index.html` الأصلي، `make-license.js`، `build.js`، ملف `.env` بتاعك.

## ب) خطوات التشغيل على سيرفر الشركة

1. **تثبيت Node.js 18+** على السيرفر.
2. فك الحزمة في مجلد، ثم:
   ```
   npm install --production
   ```
3. **قاعدة البيانات**: الشركة تنشئ مشروع مجاني على supabase.com →
   SQL Editor → لصق محتوى `supabase-schema.sql` → Run.
4. نسخ `.env.example` إلى `.env` وملء القيم:
   - `SUPABASE_URL` و `SUPABASE_ANON_KEY` من لوحة Supabase
   - `SA_USER_ID` / `SA_PIN` — حساب السوبر أدمن الخاص بالشركة
   - `JWT_SECRET` — أي نص عشوائي طويل
5. التشغيل:
   ```
   npm start
   ```
   وللتشغيل الدائم (يُنصح به):
   ```
   npm install -g pm2
   pm2 start server.js --name instaport
   pm2 save && pm2 startup
   ```
6. الدخول من المتصفح على `http://<عنوان-السيرفر>:7434`

## ج) الترخيص

- النسخة المسلّمة **مقفولة على دومينات/عناوين الشركة** المتفق عليها + localhost.
- لو اتنقلت لسيرفر أو دومين غير مرخص → تظهر شاشة "License Required" ولا يعمل التطبيق.
- تجديد/تعديل الترخيص: يتم من المورّد فقط (توليد مفتاح جديد وبناء نسخة جديدة).

### توليد ترخيص جديد (عندك أنت فقط):
```
node make-license.js "company.com,192.168.1.50" 2027-12-31 "Company Name"
node build.js --license <KEY>
```

## د) النسخ الاحتياطي

- من داخل التطبيق: **Admin → Settings → Backup & Restore → Download Full Backup**
- يُنصح بجدولة تصدير أسبوعي وحفظ الملف خارج السيرفر.
- قاعدة بيانات Supabase لها نسخ احتياطي تلقائي من لوحة التحكم أيضًا.
